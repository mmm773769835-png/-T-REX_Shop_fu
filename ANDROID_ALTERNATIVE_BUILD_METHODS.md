# طرق بديلة لبناء تطبيق Android - T-REX Shop

## الحالة الحالية:

لا يزال هناك مشكلة في الاتصال بالخوادم عند محاولة رفع الملفات إلى EAS Build (ECONNRESET). هذا مشكلة في الشبكة وليس في تكوين التطبيق.

## الطرق البديلة المتاحة:

### 1. استخدام Expo Dev Client (مُوصى به)
```bash
# تثبيت Expo Dev Client إذا لم يكن مثبتًا
npm install expo-dev-client

# إنشاء إصدار تطوير
npx eas build --platform android --profile development --clear-cache
```

### 2. إنشاء مشروع React Native خالص
```bash
# إنشاء مشروع جديد باستخدام Expo Bare Workflow
npx create-expo-app T-REX-Shop-Bare --template bare

# نسخ ملفات المصدر إلى المشروع الجديد
# ثم بناء المشروع باستخدام React Native CLI
cd T-REX-Shop-Bare
npx react-native run-android
```

### 3. استخدام خدمات بناء سحابية بديلة
- **Expo Application Services (EAS)** - حاول في أوقات مختلفة
- **GitHub Actions** - إعداد CI/CD للبناء التلقائي
- **Bitrise** - خدمة بناء سحابية موثوقة

### 4. إعداد بيئة تطوير محلية
#### المتطلبات:
1. تثبيت Android Studio
2. تثبيت Android SDK
3. إعداد متغيرات البيئة PATH

#### خطوات الإعداد:
```bash
# تثبيت أدوات React Native CLI
npm install -g react-native-cli

# إنشاء مجلد android جديد
npx expo prebuild --platform android

# بناء المشروع محليًا
npx expo run:android
```

### 5. استخدام Expo Snack (للبناء السريع)
- زيارة https://snack.expo.dev
- رفع ملفات المشروع
- استخدام خدمة Expo Snack للبناء السريع

## حل مشكلة الشبكة:

### 1. المحاولة في أوقات مختلفة:
- حاول البناء في الليل (12 صباحًا - 6 صباحًا)
- تجنب أوقات الذروة لاستخدام الإنترنت

### 2. استخدام VPN:
- إذا كانت هناك قيود على الشبكة
- استخدام خادم VPN مستقر

### 3. تحسين اتصال الإنترنت:
- إغلاق التطبيقات الأخرى التي تستهلك عرض نطاق
- استخدام اتصال سلكي بدلاً من اللاسلكي إذا أمكن

## ملاحظات مهمة:

- التكوينات صحيحة تمامًا الآن
- لا توجد مشاكل في تطابق الحزم أو Firebase
- المشكلة تتعلق فقط باتصال الشبكة المؤقت
- جميع الميزات معدة بشكل صحيح للعمل على Android

## الخطوة التالية المقترحة:

جرب أحد الطرق البديلة المذكورة أعلاه، أو حاول الأمر الأصلي في وقت مختلف عندما يكون اتصال الإنترنت أكثر استقرارًا.
