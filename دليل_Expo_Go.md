## 🧪 **اختبار المتغيرات:**

### 1️⃣ **اختبار المزامنة التلقائية:**

```
الخطوات:
1. افتح التطبيق على هاتفين مختلفين
2. سجّل دخول كمدير على الأول
3. أضف منتج جديد
4. راقب الهاتف الثاني ← سيظهر المنتج فوراً! 🔄
```

**النتيجة المتوقعة:**
- ⏱️ وقت الاستجابة: < 1 ثانية
- 📦 تحديث فوري بدون refresh
- ✅ مزامنة كاملة

---

### 2️⃣ **اختبار الحماية الأمنية:**

```
السيناريو 1: محاولة إضافة منتج بوصف يحتوي على <script>
النتيجة: ❌ سيتم رفضه (XSS Protection)

السيناريو 2: إدخال بيانات غير صحيحة في النموذج
النتيجة: ❌ رسائل خطأ واضحة (Input Validation)

السيناريو 3: محاولة وصول بدون Token
النتيجة: ❌ مرفوض (JWT Auth)
```

## 🔍 **استكشاف الأخطاء:**

### المشكلة 1: "Couldn't connect to server"

**الحل:**
```bash
# تأكد من تشغيل السيرفر
cd server
node index.js

# يجب أن ترى:
🔐 T-REX Auth Server listening on port 4001
```

---

### المشكلة 2: "Network response timed out"

**الحل:**
```bash
# تأكد من اتصال الهاتف بنفس الشبكة WiFi
# تحقق من IP الصحيح:
ipconfig | findstr /i "IPv4"

# يجب أن يكون: 172.20.44.16
```

---

### المشكلة 3: "Invalid admin credentials"

**الحل:**
```bash
# تأكد من البيانات:
البريد: mmm773769835@gmail.com
كلمة المرور: 733770042Aa

# إذا لم تنجح، أعد إنشاء كلمة المرور:
cd server
node set-admin-password.js "733770042Aa"
```

---

### المشكلة 4: "Firebase error"

**الحل:**
```bash
# تأكد من وجود المستخدم في Firebase Console:
Firebase Console → Authentication
ابحث عن: mmm773769835@gmail.com

# إذا لم يكن موجوداً، أضفه:
Add User → mmm773769835@gmail.com / 733770042Aa
```
