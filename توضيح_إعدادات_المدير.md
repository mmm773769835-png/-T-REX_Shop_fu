**المكان:** `server/.env`

**مثال في الكود:**
```javascript
// server/index.js
const adminEmail = process.env.ADMIN_EMAIL || 'mmm712874799@gmail.com';
const adminHash = process.env.ADMIN_HASH;

app.post('/api/auth/admin-login', async (req, res) => {
  const { email, password } = req.body;
  
  if (email !== adminEmail) {
    return res.status(401).json({ error: 'unauthorized' });
  }
  
  const ok = await bcrypt.compare(password, adminHash);
  if (!ok) {
    return res.status(401).json({ error: 'unauthorized' });
  }
  
  // Generate JWT token...
});
```

## 🔄 **كيف يعمل النظام معاً؟**

### سيناريو 1: تسجيل دخول عبر Firebase مباشرة

```
1. المستخدم يدخل البريد وكلمة المرور
   ↓
2. Firebase يتحقق من البيانات
   ↓
3. Firebase يرسل ID Token
   ↓
4. التطبيق يستخدم Token للوصول لـ Firestore
   ↓
5. Firestore Rules يتحقق من البريد
   ↓
6. ✅ السماح بالصلاحيات
```

---

### سيناريو 2: تسجيل دخول عبر Server API

```
1. المستخدم يدخل البريد وكلمة المرور
   ↓
2. Server يتحقق من .env (ADMIN_EMAIL + ADMIN_HASH)
   ↓
3. Server يولد JWT Token
   ↓
4. التطبيق يحفظ Token في AsyncStorage
   ↓
5. عند كل طلب، يُرسل Token مع الـ Request
   ↓
6. Server يتحقق من Token ويرسل البيانات
   ↓
7. ✅ نجاح العملية
```

### 3️⃣ **Firestore Rules**

**الملف:** `firestore.rules`

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    function isLoggedIn() {
      return request.auth != null;
    }
    
    function isAdmin() {
      return request.auth != null && 
             request.auth.token.email == 'mmm712874799@gmail.com';
    }
    
    match /products/{productId} {
      allow read: if true;
      allow create, update, delete: if isAdmin();
    }
  }
}
```

**لتغيير البريد في القواعد:**
1. افتح `firestore.rules`
2. غيّر البريد في دالة `isAdmin()`
3. احفظ الملف
4. انشر القواعد:
```bash
firebase deploy --only firestore:rules
```
