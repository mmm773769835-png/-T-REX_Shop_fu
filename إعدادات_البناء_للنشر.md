# إعدادات البناء للنشر - T-REX Shop

## 📋 نظرة عامة

هذا الدليل الشامل يغطي جميع إعدادات البناء والنشر لتطبيق T-REX Shop على منصات Android و iOS.

---

## 🔧 الإعدادات الأساسية

### 1. إعدادات EAS Build (`eas.json`)

تم تحسين ملف `eas.json` ليشمل إعدادات متقدمة للنشر:

#### **Development (التطوير)**
- **النوع:** APK للاختبار الداخلي
- **الاستخدام:** للاختبار المحلي والتطوير
- **الأمر:**
  ```bash
  eas build --platform android --profile development
  ```

#### **Preview (معاينة)**
- **النوع:** APK مُحسّن للاختبار الخارجي
- **الاستخدام:** مشاركة مع المختبرين والمراجعين
- **الأمر:**
  ```bash
  eas build --platform android --profile preview
  ```

#### **Production (إنتاج)**
- **النوع:** App Bundle (AAB) للنشر على Google Play
- **الاستخدام:** النشر الرسمي على متاجر التطبيقات
- **الأمر:**
  ```bash
  eas build --platform android --profile production
  ```

## 📱 إعدادات Android

### معلومات التطبيق الأساسية

```json
{
  "package": "com.trexshop.app",
  "version": "2.0.0",
  "versionCode": 2,
  "minSdkVersion": 21,
  "targetSdkVersion": 34,
  "compileSdkVersion": 34
}
```

### الأذونات (Permissions)

```xml
- INTERNET (الإنترنت)
- VIBRATE (الاهتزاز)
- READ_EXTERNAL_STORAGE (قراءة التخزين)
- WRITE_EXTERNAL_STORAGE (كتابة التخزين)
- CAMERA (الكاميرا)
```

### إعدادات البناء (build.gradle)

#### **Optimization Settings (إعدادات التحسين)**

في ملف `android/gradle.properties`:

```properties
# تحسين الذاكرة
org.gradle.jvmargs=-Xmx2048m -XX:MaxMetaspaceSize=512m

# البناء المتوازي (أسرع)
org.gradle.parallel=true

# دعم AndroidX
android.useAndroidX=true

# ضغط الصور في بناء الإنتاج
android.enablePngCrunchInReleaseBuilds=true

# معماريات مدعومة
reactNativeArchitectures=armeabi-v7a,arm64-v8a,x86,x86_64

# محرك JavaScript: Hermes (أسرع وأصغر)
hermesEnabled=true

# دعم GIF و WebP
expo.gif.enabled=true
expo.webp.enabled=true
```

#### **Release Build Configuration (إعدادات بناء الإنتاج)**

في `android/app/build.gradle`:

```gradle
buildTypes {
    release {
        // تفعيل تقليل حجم التطبيق
        minifyEnabled true
        shrinkResources true
        
        // استخدام ProGuard للتحسين
        proguardFiles getDefaultProguardFile("proguard-android-optimize.txt"), "proguard-rules.pro"
        
        // ضغط الصور
        crunchPngs true
        
        // ⚠️ ملاحظة مهمة: يجب إنشاء keystore للتوقيع في الإنتاج
        // signingConfig signingConfigs.release
    }
}
```

## 🔐 إعدادات التوقيع (Signing)

### Android - Keystore للتوقيع

⚠️ **مهم جداً:** يجب إنشاء keystore خاص للتوقيع في بناء الإنتاج.

#### **إنشاء Keystore للإنتاج:**

```bash
keytool -genkeypair -v -storetype PKCS12 -keystore trexshop-release.keystore -alias trexshop-key-alias -keyalg RSA -keysize 2048 -validity 10000
```

#### **إضافة Keystore إلى EAS:**

1. إنشاء Keystore على جهازك
2. رفع Keystore إلى EAS:
   ```bash
   eas credentials
   ```
3. اتبع التعليمات لإضافة بيانات التوقيع

#### **إعدادات التوقيع في build.gradle:**

```gradle
signingConfigs {
    release {
        if (project.hasProperty('MYAPP_RELEASE_STORE_FILE')) {
            storeFile file(MYAPP_RELEASE_STORE_FILE)
            storePassword MYAPP_RELEASE_STORE_PASSWORD
            keyAlias MYAPP_RELEASE_KEY_ALIAS
            keyPassword MYAPP_RELEASE_KEY_PASSWORD
        }
    }
}
```

### iOS - Certificates و Profiles

يتم إدارة شهادات iOS تلقائياً عبر EAS عند استخدام:
```bash
eas credentials
```

```gradle
defaultConfig {
    versionCode 2
    versionName "2.0.0"
}
```

## 🎯 التحسينات المتقدمة

### 1. تقليل حجم التطبيق

في `android/gradle.properties`:
```properties
# تفعيل ضغط الموارد
android.enableShrinkResourcesInReleaseBuilds=true

# تفعيل تقليل الكود
android.enableMinifyInReleaseBuilds=true
```

### 2. ProGuard Rules

في `android/app/proguard-rules.pro`:
```proguard
# احتفظ بقواعد Firebase و React Native
-keep class com.facebook.react.** { *; }
-keep class com.google.firebase.** { *; }
```

### 3. Split APKs (تقسيم APK حسب المعمارية)

```gradle
android {
    splits {
        abi {
            enable true
            reset()
            include "armeabi-v7a", "arm64-v8a", "x86", "x86_64"
            universalApk true
        }
    }
}
```

## 📦 النشر على المتاجر

### Google Play Store

#### **خطوات النشر:**

1. **إنشاء App Bundle:**
   ```bash
   eas build --platform android --profile production
   ```

2. **تحميل AAB إلى Google Play Console:**
   - اذهب إلى [Google Play Console](https://play.google.com/console)
   - اختر تطبيقك
   - اذهب إلى "Production" > "Create new release"
   - ارفع ملف `.aab` الذي تم إنشاؤه

3. **ملء معلومات الإصدار:**
   - رقم الإصدار
   - ملاحظات الإصدار (Release Notes)
   - الصور والمعلومات المطلوبة

4. **المراجعة والنشر:**
   - راجع جميع المعلومات
   - اضغط "Review release"
   - ثم "Start rollout"

### Apple App Store

#### **خطوات النشر:**

1. **إنشاء Build:**
   ```bash
   eas build --platform ios --profile production
   ```

2. **الرفع إلى App Store Connect:**
   ```bash
   eas submit --platform ios
   ```
   أو يدوياً عبر Xcode

3. **إعداد App Store Connect:**
   - معلومات التطبيق
   - الصور والوصف
   - معلومات التسعير

4. **إرسال للمراجعة:**
   - أكمل جميع المعلومات
   - أرسل للمراجعة من Apple
