# 📋 دليل خطوة بخطوة: استخدام سيرفر موجود في مجلد آخر

## 🎯 الخطوات السريعة

### الخطوة 1: افتح ملف الإعدادات

افتح الملف: `config/api.config.js`

### الخطوة 2: حدّث القيم حسب سيرفرك

#### الحالة 1: السيرفر على الإنترنت (Production)

```javascript
const USE_LOCAL_SERVER = false;
const PRODUCTION_API_URL = "https://your-actual-server.com";
```

**مثال:**
```javascript
const PRODUCTION_API_URL = "https://api.myshop.com";
```

#### الحالة 2: السيرفر على جهاز آخر في نفس الشبكة

```javascript
const USE_LOCAL_SERVER = true;
const USE_IP = true;
const LOCAL_IP = '192.168.1.100'; // IP address الجهاز الذي يعمل عليه السيرفر
const PORT = 3000; // المنفذ
```

**مثال:**
- إذا كان السيرفر يعمل على جهاز IP: `192.168.1.50` وعلى المنفذ `8080`
```javascript
const LOCAL_IP = '192.168.1.50';
const PORT = 8080;
```

#### الحالة 3: السيرفر على نفس الجهاز (localhost)

```javascript
const USE_LOCAL_SERVER = true;
const USE_IP = false;
const USE_LOCALHOST = true;
const PORT = 3000;
```

## 🔍 كيفية معرفة عنوان السيرفر

### إذا كان السيرفر على الإنترنت:
- عنوان URL الكامل (مثلاً: `https://api.example.com`)
- لا يحتاج إلى منفذ في معظم الأحيان

### إذا كان السيرفر محلياً:

#### 1. معرفة IP address الجهاز:
- افتح PowerShell/CMD
- اكتب: `ipconfig`
- ابحث عن "IPv4 Address"

#### 2. معرفة المنفذ (Port):
- افتح ملف السيرفر (مثلاً: `index.js` أو `app.js`)
- ابحث عن: `app.listen(XXXX)` أو `PORT`
- الرقم هو المنفذ (مثلاً: 3000, 8080, 5000)

## ✅ التحقق من أن كل شيء يعمل

### 1. تحقق من الإعدادات

عند تشغيل التطبيق، ستظهر في console:
```
🔗 API Server URL: http://...
```

تأكد أن العنوان صحيح!

### 2. اختبر الاتصال

افتح المتصفح واذهب إلى:
```
http://YOUR_SERVER/api/auth/check
```

يجب أن ترى: `{"ok":false}` (هذا طبيعي - يعني السيرفر يعمل)

### 3. جرّب في التطبيق

- افتح التطبيق
- جرّب تسجيل الدخول
- يجب أن يعمل OTP الآن!

## 🆘 مشاكل شائعة

### المشكلة: "Network Error"

**الحل:**
1. ✅ تأكد أن السيرفر يعمل
2. ✅ تحقق من العنوان في `config/api.config.js`
3. ✅ اختبر الاتصال في المتصفح

### المشكلة: "Cannot connect"

**الحل:**
1. ✅ تأكد أن الهاتف والكمبيوتر على نفس الشبكة (إذا كان محلي)
2. ✅ تحقق من Firewall - قد يحتاج السيرفر لإذن

### المشكلة: "CORS Error"

**الحل:**
- في ملف السيرفر، تأكد من وجود:
```javascript
const cors = require('cors');
app.use(cors());
```

## 📝 ملاحظات مهمة

1. **للهاتف الحقيقي:** استخدم IP address وليس localhost
2. **للمحاكي Android:** استخدم IP address
3. **للمحاكي iOS:** يمكن استخدام localhost
4. **للسيرفر الخارجي:** استخدم HTTPS في الإنتاج

## 💡 مثال كامل

لنفترض أن لديك سيرفر يعمل على:
- الجهاز: `192.168.1.100`
- المنفذ: `3000`

في `config/api.config.js`:
```javascript
const USE_LOCAL_SERVER = true;
const USE_IP = true;
const LOCAL_IP = '192.168.1.100';
const PORT = 3000;
```

ستصبح API URL: `http://192.168.1.100:3000`
